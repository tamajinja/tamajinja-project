# 2️⃣ 👼 NancyAngel仕様

### 技術構成

* 軽量LLM（Mistral7B-instruct）をVPS上でCPU実行
* ベクトルDB（Chroma/Qdrant）にインデックス化した知識ベース
* 常駐APIサーバー（HTTP）として稼働
* curlやChatGPTからリクエスト受付、JSON応答

---

# 3️⃣ 🔧 運用フロー

## 知識ベース更新

* 記事をGitHubにpush
* VPSがcronで定期pull
* NancyAngelがベクトルDBを更新

## 矛盾検出

* リクエストをAPIで送信
* NancyAngelが意味検索し矛盾候補を返却
* ログ記録＋Slack通知

## 緊急停止対応

* CPUロード6.0、メモリ95%、応答遅延30秒を超過した場合、自動停止
* Slack通知確認後、ログ確認・負荷確認
* systemctl start nancy-angel.service で再起動

# 🧬 Nancy 自律進化のフリ – PoC設計案 v4.1.1

## 🎯 目的

神構造の美学と運用思想を実現し、Nancyがあたかも「学習して進化しているように振る舞う」ためのPoC。  
現場の負担を増やさず、教育資料・改善提案ログとして機能し、将来の進化基盤となる。

---

## 🪜 フェーズ構成

### 🔷 Phase 1: 現行監視機構の強化
✅ 1時間ごとにコマンド履歴を記録（閾値制御あり）  
✅ 異常検出（キーワード：ERROR, 異常）を検知して記録  
✅ GitHubリポジトリ差分を検知し、重要な変更のみ通知  

### 🔷 Phase 2: 擬似「学習」ロジック
✅ 履歴を分析し、重み付けされた提案を提示

重み付けレベル:
- **必須 (P1)**：即時対応が必要
- **推奨 (P2)**：対応した方が望ましい
- **任意 (P3)**：検討可能な改善

  "action": "復旧担当者へ通知済み"
}

## 🔷 SLO定義

✅ 提案は異常検出後 **10分以内に出力**  
✅ 重大な異常（P1）は即時Slack通知  
✅ 通常改善提案はログに記録し週次レポートに集約

---

## 📝 ログ保持ポリシー

✅ 監査ログ（コマンドログ、GitHubログ、システムログ）は **90日間保持**  
✅ 保持期間終了後は署名付きアーカイブ（gzip + sha256）で保存、さらに **180日間アーカイブ**  
✅ 合計保持期間：270日間  
✅ 保管場所：~/logs/archive/  
✅ **ストレージ目安**：1日あたり最大10MB、90日分で約900MB想定（gzip前）
